🚀 WebOS浏览器自动代理功能

✨ 新功能：智能自动代理

现在浏览器会自动检测外部网站并通过代理访问，无需手动添加代理前缀！

只需要输入正常的网址，系统会自动处理代理访问。

🔧 功能特性

🤖 智能检测

自动识别外部网站（非localhost）并使用代理访问

🎯 用户友好

地址栏显示原始URL，用户体验更自然

🔄 可视化指示器

状态栏显示当前是否使用代理

⚡ 无缝切换

本地网站直接访问，外部网站自动代理

📋 使用说明

  1. 打开WebOS浏览器应用
  2. 在地址栏直接输入网址，例如：www.baidu.com
  3. 系统自动检测并使用代理访问
  4. 观察状态栏的代理指示器
  5. 享受无障碍的网页浏览体验

🎨 状态指示器说明

🔄
代理模式 - 正在通过代理服务器访问外部网站
用于访问被X-Frame-Options限制的网站
🔗
直接模式 - 直接访问本地或兼容的网站
用于访问localhost或允许iframe嵌入的网站

💡 技术原理

智能检测算法：

  • 检查URL是否包含 localhost 或 127.0.0.1
  • 检查URL是否已经是代理地址
  • 对于外部URL，自动转换为：
    http://localhost:9000/proxy?url=原始网址
  • 保持地址栏显示原始URL，用户无感知

重要提醒：

  • 代理服务器必须运行在端口9000
  • 首次访问某些网站可能需要等待几秒钟
  • 部分网站可能有额外的安全限制

🔍 故障排除

如果网站无法加载：

  1. 检查代理服务器是否运行：访问 http://localhost:9000
  2. 查看浏览器状态栏的代理指示器
  3. 等待页面完全加载（某些网站较慢）
  4. 刷新页面重试
